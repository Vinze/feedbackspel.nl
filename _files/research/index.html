Dit soort applicaties luisteren actief naar gebeurtenissen en verwerken deze informatie vrijwel direct. Deze gebeurtenissen kunnen plaatsvinden op een (externe) server of geleverd worden door een service. Een voorbeeld van een aantal realtime applicaties zijn bijvoorbeeld chatprogramma's, bluetooth software, besturingssystemen en multiplayer games. ## Geschiedenis Software geschreven in bijvoorbeeld de programmeertaal C kan al jaren lang gebruik maken van realtime data. Doordat dit soort programma's zeer dicht bij de hardware staan kan het communiceren met de componenten in de computer, en deze informatie soms wel duizenden keren per seconde bijwerken. Sinds de laatste jaren wordt ook het gebruik van realtime data op internet steeds belangrijker, en diverse ontwikkelaars hebben programma's gemaakt waarmee dit mogelijk wordt. Hieronder worden enkele technieken besproken welke toegepast kunnen worden en welke voor en nadelen hieraan kleven. ### HTTP polling
De client stuurt op een vaste interval een verzoek (rood) naar de server om te controleren of er nieuwe data beschikbaar is. De server stuurt hierop een antwoord (blauw) terug. Indien er nieuwe data (geel) beschikbaar is zal de server deze data bij het volgende verzoek terugsturen. Vervolgens ontvangt de client de data en kan dit verder verwerken. Door de interval erg laag te zetten, bijvoorbeeld om de 50 milliseconde, kan de gebruiker dit ervaren als realtime.
![Polling afb.](img/polling.png)
### HTTP long-polling
Long-polling lijkt erg veel op polling met het verschil dat de verbinding bij long-polling tijdelijk open blijft staan. De client doet een verzoek (rood) naar de server om te controleren of er nieuwe data (geel) beschikbaar is. De server wacht met het geven van antwoord tot nieuwe data beschikbaar is. Zodra dit het geval is stuurt de server deze data naar de client (blauw) en herhaalt dit proces zich weer.
![Long-polling afb.](img/long-polling.png)
### HTTP streaming
Streaming lijkt enigzins op long-polling met het grootste verschil dat de verbinding nu vanuit de server open blijft staan. De client doet eenmalig een verzoek tot informatie (rood), en de server zal deze data (geel) in gedeeltes opsturen naar de client (rood).
![Streaming afb.](img/streaming.png)
http://www.w3.org/TR/eventsource/ ### Websockets
Moderne webbrowsers kunnen tegenwoordig ook gebruikmakan van het in 2011 gestandaardiseerde ([bron](http://tools.ietf.org/html/rfc6455)) websocket protocol. In tegenstelling tot bovenstaande technieken is het mogelijk om met websockets een zogenaamde 'full-duplex' verbinding op te zetten. Dit houd in dat zowel de client als de server tegelijkertijd data naar elkaar kunnen sturen. De client doet een verzoek tot verbinding (rood) met de server. Indien alles goed gaat zal de server deze verbinding accepteren (blauw) en komt er een nieuwe full-duplex verbinding tot stand waarover gegevens verzonden kunnen worden.
